# 2.5D Asset Replacement Guide

## Goal

Make externally produced assets easy to drop into the Phaser client without touching gameplay code.

The current game can run with generated fallback textures. Production assets should replace those textures by using the same stable asset keys.

## Current Structure

- `src/client/game/assets/manifest.ts`
  Defines stable runtime keys and intended external file paths.

- `src/client/game/assets/generated.ts`
  Generates fallback textures in Phaser when real image files are not enabled.

- `src/client/game/assets/loader.ts`
  Owns the switch between external image loading and generated fallback textures.

- `public/assets/raceway/`
  Target folder for externally produced PNG files.

## Replacement Rule

Do not change gameplay code when replacing art.

Instead:

1. Export PNG files into `public/assets/raceway/`.
2. Match the filenames listed in `EXTERNAL_IMAGE_ASSETS`.
3. Change `USE_EXTERNAL_ASSETS` in `src/client/game/assets/loader.ts` to `true`.
4. Run `npm run build`.

## Art Direction

Target a `2.5D toy raceway arena` look.

Use these rules:

- transparent PNG backgrounds
- bottom-center visual anchor
- light from upper-left
- shadow falling down-right
- readable silhouette at `48px` tile scale
- no baked text labels in character, bomb, or flame sprites
- power-up symbols may include simple readable letters or icons

## Sprite Anchoring

The Phaser scene currently places entities by tile center.

For easy replacement:

- keep the visual center near the PNG center
- keep the contact shadow near the lower third
- avoid large transparent padding differences between related frames
- keep player frames the same canvas size

## Suggested AI Prompt Shape

Use this for first-pass image generation:

```text
Create a transparent-background 2.5D toy raceway game sprite.
Subject: [player / bomb / flame / power-up].
Style: colorful toy-board arcade, chunky rounded shapes, upper-left light, soft down-right shadow, readable at 48px tile scale.
Camera: top-down 2.5D, slightly elevated, no perspective distortion.
Canvas: centered object, no scenery, no text, no UI, no border.
Output: single game-ready PNG sprite.
```

For animation strips, keep one approved seed frame and generate the strip from it. Do not generate each frame independently.
